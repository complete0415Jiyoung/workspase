1. 인터페이스란?
  interface : 접점
- 연관성이 낮거나 없는 클래스들에게 공통된 기능을 제공할때 사용 

	ex) 키보드, 마우스,스캐너, 카메라, 압력센서, 기울기 센서
	--> 입력 장치(각각의 용도는 다르지만 "입력"이라는 공통된 "기능 명"이 있을 뿐)

	+ 모든 필드(변수)가 묵시적 (암묵적) public static final
	ex) public static final double p1 = 3.141592;
  	  /*public static final*/ int MAX_NUM = 10000;

	+ 모든 메소드가 묵시적으로 public abstract (추상메소드)

	ex) /*public abstract*/ void input();// "input"이라는 이름을 자식에게 제공할 뿐
			//상세한 기능은 자식이 알아서 오버라이딩 해라 
			//그런데 추상메소드니까 오버라이딩 강제화!

2. 자바에서 컬렉션이란?

[컬렉션] : 자바에서 제공하는 자료구조를 담당하는 프레임 워크 (틀)
	- 삭제, 수정, 추가 등 기능처리을 간단하게 해결하여 
	  자료 구조의 알고리즘을 구현할 필요가 없다. 

3. try - catch -finally 에 대해 설명하시오
try에서 발생한 코드를 catch로 던지고 예외를 처리하고 수행하여 발생하는 오류가 없도록 함.

try{ 예외가 발생할 가능성이 있는 수행코드
}catch(예외){ 예외 발생시 수행할 코드
}finally{예외가 발생하든 말든 무조건 실행하는 코드}

4. 향상된 for문 작성법
for(하나씩 꺼내서 저장할 변수 : 배열 또는 컬렉션) { 수행구문 }

5. 컬렉션 인터페이스란?

collection 인터페이스 : List 와 Set의 공통 코드를 작성한 단순한 부모 인터페이스 / add
* List:  인덱스가 있는 배열의 형태 
		- 순서 유지하고 저장 가능
		- 중복 저장 가능 
		- 구현 클래스 : ArrayList / LinkedList

* Set : 주머니 형태
		- 순서 유지하지 않고 저장
		- 중복 저장 안됨(null도 중복 안됨)
		- 중복 객체가 들어 올때 이전 객체 삭제  
		- 구현 클래스 : HashSet / LikedeList / TreeSet
