08_inheritence(상속) 

<상속> 
: 다른 클래스에 있는 멤버 (필드, 메소드)를 
 새로운 클래스에 새로 만들지 않고 상속 받음으로써
 새 클래스가 자신의 멤버처럼 사용하게하는 기술 

* 목적: 클래스의 재사용 연관된 		
         일렬의 클래스에 대한 공통적인규약 정리 
	
* 장점
	1) 보다 작은 양의 코드로 새로운 클래스 작성 가능
	2) 코드를 공통적으로 관리하기 떄문에 코드의 추가 및 변경이 용이 
	3) 코드의 중복을 제거하여 프로그램 생산성과 유지보수에 기여 
	
* 자바는 다중 산속은 미지원하고 단일 상속만 지원 

*상속의 특징 
	1) 모든 클래스는 Object 의 후손이다.
		Object 클래스의 메소드를 오버라이딩하여 재구현이 가능하다 
	2) 부모 클래스 생성자, 초기화 블록은 상속이 되지 않는다
		super()사용해서 상속 가능 		
	3) 부모 private 멤버는 상속이 되지만 직접 접근이 불가능하다 
		this. name = name; (x)
		setName(name); (o)
		super ( name ); (o)

* suer() :  부모 객체의 생성자를 호출하는 메소드 
	 기본적으로 후손 생성에 부모 생성자를 포함한다. 
	 후손 객체 생성시에는 부모로 부터 생성되기 때문에 	
	 후손클래스 생성자 안에는 부모 생성자를 호출하는
	 super()가 첫출에 존재해야한다 

* super. : 자식클래스에 저의 시 부모 객체를 가르키는 참조 변수 
		자식클래스에서 부모 클래스 객체에 접근하여 
		필드나 메소드 호출시 사용한다.

*오버라이딩(Overriding) 
	: 자식 클래스가 상속 받은 보모의 메소드를 재정의 한 것 

	@overide - 오버라이드 어노테이션
	: 컴파일러에게 오버라이딩 되었음을 알려주는 인식용 주석 
	
	[성립조건]
	1) 메소드 이름이 동일해야 한다 
	2) 변환형이 동일해야한다 
	3) 매개 변수가 동일해야한다 
	4) 접근제한자가 같거나 넓은 범위여야한다 
	5) 예외처리 범위가 같거나 좁아야한다 
		+ pivate 메서드는 오버라이딩 불가 

* 예약어 final 
	final 클래스 : 상속이 불가능한 클래스
	final 메소드 : 상속시 오버라이딩이 불가능한 메소드
